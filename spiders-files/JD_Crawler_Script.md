## 功能简介

本爬虫基于 **DrissionPage** 模拟浏览器操作，通过监听京东评论 API 响应来抓取商品评论数据。

核心特性：

- 🔄 **多店铺批量爬取**：自动遍历配置的多个商品链接
- 💾 **断点续传**：重启脚本自动加载历史数据，跳过已抓取的评论
- 🛡️ **人机验证检测**：自动识别京东验证弹窗，暂停等待手动通过
- 🧠 **拟人化滚动**：随机混合空格键/大小幅滚动，降低被风控识别的概率
- ⚡ **实时增量保存**：每批数据立即写入文件，页面崩溃也不丢数据
- 🔑 **登录态持久化**：使用 User Data Dir 保持京东登录状态，避免每次重新登录

## 环境安装

### 1. Python 版本

需要 Python **3.6+**

### 2. 安装依赖

```bash
pip install DrissionPage
```

> DrissionPage 是唯一的第三方依赖，它会自动管理浏览器驱动，无需手动下载 ChromeDriver。

### 3. 浏览器要求

需要已安装 **Chromium 内核浏览器**（Chrome / Edge 均可），DrissionPage 会自动检测。

## 首次使用（登录）

> ⚠️ **首次运行必须完成此步骤！** 脚本依赖京东登录态来降低风控概率。

### 步骤

1. **直接运行脚本**：

   ```bash
   python JD_Crawler_Script.py
   ```

2. 脚本会打开一个浏览器窗口并访问第一个商品页面
3. 在浏览器中**手动登录京东账号**（推荐使用京东 APP 扫码登录）
4. 登录成功后，脚本会在项目目录生成 `JD_User_Data/` 文件夹，保存你的登录信息
5. **此后每次运行，浏览器会自动保持登录状态，无需再次登录**

> 💡 **提示**：如果登录过期了，删除 `JD_User_Data/` 文件夹，重新运行脚本登录即可。

## 使用方法

### 基本用法

```bash
python JD_Crawler_Script.py
```

脚本没有命令行参数，所有配置在脚本文件头部的 **配置区域** 修改。

### 运行流程

```
1. 启动浏览器 → 加载历史数据（断点续传）
2. 依次访问每个商品URL
3. 自动打开评论弹窗（失败时会提示你手动点击）
4. 循环滚动评论区 → 监听API响应 → 解析并保存数据
5. 连续10轮无新数据 → 判定本店已抓完 → 切换下一家
6. 全部店铺爬完 → 最终保存并退出
```

## 配置说明

打开 `JD_Crawler_Script.py`，修改文件顶部的配置区域：

```python
# ================= ⚙️ 配置区域 =================
PRODUCT_NAME = "HuaWei P70 "       # 产品名称标签
SHOP_URLS = [                       # 商品链接列表
    "https://item.jd.com/xxx.html",
    "https://item.jd.com/yyy.html",
]
SCROLL_TIMES = 5000                 # 每个店铺最大滚动轮次
JSON_FILE = 'data/jd_HuaWei_P70.json'  # 数据保存路径
USER_DATA_DIR = './JD_User_Data'    # 浏览器用户数据目录
# ===============================================
```

### 参数详解

| 参数 | 说明 | 建议 |
|------|------|------|
| `PRODUCT_NAME` | 写入每条数据的产品名称标签 | 改成你要爬的产品名 |
| `SHOP_URLS` | 京东商品页面链接列表 | 支持不同店铺的同款商品 |
| `SCROLL_TIMES` | 每个店铺最多滚动多少轮 | 默认5000轮足够，一般会先触发连续10轮无数据自动跳过 |
| `JSON_FILE` | 数据保存的JSON文件路径 | 换产品时记得改文件名 |
| `USER_DATA_DIR` | 浏览器登录态存储目录 | 一般不需要改 |

### 如何添加新商品链接

1. 在京东搜索你要爬的商品
2. 点进不同店铺的商品详情页
3. 复制地址栏的URL（格式如 `https://item.jd.com/数字.html`）
4. 添加到 `SHOP_URLS` 列表中

## 防封与反检测机制

本脚本内置了多层防封策略，正常使用时**不需要你手动干预**：

| 策略 | 说明 |
|------|------|
| 🔑 登录态持久化 | 保持京东登录状态，风控阈值比游客高很多 |
| 🎯 精确监听 | 只监听 `api.m.jd.com/client.action`，减少无关请求噪声 |
| 🧠 拟人化滚动 | 随机混合空格翻页、小幅/中幅滚动，避免规律性操作 |
| ⏱️ 自适应休眠 | 正常 12-20s，空轮次 15-25s，长休眠 45-90s |
| 🛡️ 验证检测 | 自动识别弹窗并暂停，不会傻傻继续滚动 |
| 💾 实时保存 | 触发验证/页面刷新时数据不丢失 |

## 常见问题

### Q: 弹窗打不开怎么办？

脚本会提示 `❌ 自动打开失败，请手动点击后回车`。在浏览器里手动找到并点击 **"全部评价"** 或 **"商品评价"** 按钮，然后回到终端按回车即可。

### Q: 一直触发人机验证怎么办？

1. 检查是否已登录京东账号（首次运行需要手动登录）
2. 如果频繁触发，可能是当前 IP 已被标记，尝试：
   - 等待一段时间后再试（建议 30 分钟以上）
   - 切换网络（手机热点 / VPN）
   - 换一个京东账号

### Q: 提示"连续10轮无新包"但实际还有评论？

可能是评论区滚动到底后，剩余评论需要翻页。当前版本依赖弹窗内的滚动加载，少数情况下可能漏掉尾部数据。这部分数据量通常很小，不影响整体质量。

### Q: 怎么爬另一个商品？

修改配置区的 3 个参数：
```python
PRODUCT_NAME = "你的产品名"
SHOP_URLS = ["商品链接1", "商品链接2", ...]
JSON_FILE = 'data/jd_你的产品名.json'
```

### Q: 数据和之前的拼多多/淘宝格式一致吗？

是的。`platform`、`id`、`content`、`score`、`date`、`model_sku`、`append_content`、`votes`、`images` 这些字段在三个平台的爬虫中保持一致，可以直接合并分析。

## 注意事项

1. **保持浏览器窗口可见**：脚本通过真实浏览器操作，运行时不要最小化浏览器窗口
2. **不要手动操作浏览器**：爬取过程中请勿在浏览器中进行其他操作，以免干扰脚本
3. **Ctrl + C 随时退出**：数据会自动保存，无需担心丢失
4. **Cookie 安全**：`JD_User_Data/` 目录包含你的京东登录信息，**不要上传到 GitHub 或发给他人**
5. **合理使用**：请遵守京东平台协议，控制爬取频率，仅用于学术研究
