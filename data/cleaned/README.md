## 四、预处理流程

原始评论数据经过以下预处理步骤：

1. **删除系统默认好评**：移除各平台的默认评价模板，
   包括京东的「此用户未填写评价内容」、拼多多的「该用户觉得商品很好，给出了5星好评」等。
2. **删除单字评论**：移除有效字符 ≤1 的超短评论。
3. **追评拼接**：将追加评论（`append_content`）以句号连接拼接到原始评论末尾，作为同一条评论的延续。
4. **按评论 ID 去重**：移除具有相同平台评论 ID 的重复条目。
   拼多多平台以商品（SPU）为单位聚合评论，不同店铺展示同一评论池，因此按 ID 去重可自然消除跨店铺重复。
5. **字段标准化**：统一平台名称（`jd`/`taobao`/`pdd`）、产品名称（10 个标准名）、
   日期格式（`YYYY-MM-DD`）及 SKU 描述。移除实验不需要的字段（`votes`、`images`、`reply`、`score`）。

## 五、数据结构

```json
{
  "id": "jd_103902930162031324",
  "platform": "jd",
  "product": "iPhone_17_Pro",
  "content": "手机非常好用，拍照清晰...",
  "date": "2026-01-27",
  "sku": "岩灰色 512GB"
}
```

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `id` | string | 唯一标识符（平台前缀 + 原始评论 ID） |
| `platform` | string | 来源平台：`jd`、`taobao` 或 `pdd` |
| `product` | string | 标准化产品名称（共 10 个取值） |
| `content` | string | 评论正文（含追评拼接） |
| `date` | string | 评论日期，格式 `YYYY-MM-DD` |
| `sku` | string | 购买配置（颜色、存储容量等） |
